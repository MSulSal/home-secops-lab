# BOOTSTRAP

## Project Purpose

Build and operate a local, reproducible network security monitoring pipeline that provides DNS visibility, IDS alerting, centralized telemetry, and anomaly analysis for defensive operations.

## System Architecture Overview

Core data flow:

1. Network and DNS telemetry is generated by monitored services and hosts.
2. Zeek and Suricata produce structured logs and alert events.
3. DNS logs are exported from Pi-hole or AdGuard Home.
4. Promtail tails and labels telemetry sources.
5. Loki stores indexed log streams.
6. Grafana provides query, visualization, and alerting interfaces.
7. `ml/analysis_cli` runs explainable anomaly detection over exported telemetry features.

## Local Run

From `tooling/docker`:

```bash
docker compose up -d
```

Validation steps:

1. Confirm containers are healthy: `docker compose ps`
2. Access Grafana: `http://localhost:3000`
3. Verify Promtail targets are active
4. Query Loki labels/log streams

Shutdown:

```bash
docker compose down
```
